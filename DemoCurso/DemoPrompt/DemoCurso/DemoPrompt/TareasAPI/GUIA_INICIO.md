# Guía de inicio

Opciones para ejecutar:
1. dotnet run
2. dotnet watch run
3. Visual Studio - Ejecutar proyecto

Cómo acceder a la API: https://localhost:7091/swagger

Datos de ejemplo precargados: 3 tareas en memoria

4 formas de probar: Swagger, archivo .http, cURL, Postman

Tabla de endpoints: Ver `Controllers/TareasController.cs`

Errores comunes: Puerto en uso, certificado de desarrollo. Soluciones: cambiar launchSettings o eliminar certificado.

Configuración: Ver `appsettings.json` y `launchSettings.json`
